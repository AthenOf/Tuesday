TuesDay

visual novel engine

Tuesday бесплатный движок с открытым исходным кодом для создания визуальных новелл, интерактивных историй и текстовых приключенческих игр. Написан на языке JavaScript без использования сторонних библиотек. присутствует поддержка инфраструктуры Apache Cordova для создания нативных приложений для мобильных устройств и настольных компьютеров. Движок использует стандартные элементы Html документа, как div и img, что делает возможным использование почти любых медиа форматов поддерживаемых браузерами, включая векторную графику в формате svg, gif анимации и обширные возможности css стилей.






Структура

Отображение истории на html-странице происходит через элемент div с id tuesday. Параметры отображения можно задать в стиле css или любым другим подходящим способом. Все содержание будет отображаться с учетом параметров этого элемента.


Сценарий истории со всеми данными задается в структуре JSON. Этот независим текстовый формат обмена данными и может быть интерпретирован практически любым другим языком программирования или игровым движком. (это дает возможность переноса истории на другой движок)


Для запуска истории нужно выполнить функцию load_story('тип',файл/массив) с указанием типа данных

data: указание переменной с массивом истории из кода javascript

file: путь к json файлу с историей (может не работать локально, если в браузере включена cross domain защита)






Начало использования

Для работы подойдет любой текстовый редактор или IDE с поддержикой работы с текстовым форматом ASCII

Рекомендуемое программное обеспечения для использования

MS Windows: Notepad++

Mac OS: Xcode

	<div id='tuesday' style='width:100%; height:90vh;'></div>
		<!--Добавит тег script перед tuesday.js --> ;
		<script>
			let story = <!--вставить содержимое из story.json --> ;
		</script>
		<script type="text/javascript" src="tuesday.js"></script>
	</body>
</html>

2. Решение - В некоторых браузерах можно отключить cross domain (не рекомендуется)


FireFox - в адресной строке введите about:config в появившехся настройках найдите пункт security.fileuri.strict_origin_policy и установить ему значение false для отключения или true для включения

AutoSave


Параметр отвечает за автоматическое сохранение прогресса истории и значений переменных каждый после запуска диалога. Это позволяет выключить проект и в дальнейшем продолжить его с того же места.

	"autosave": настройка автосохранения прогресса true / false,

За продолжения с последнего места отвечает кнопка "load_autosave"

Languares


Массив поддерживаемых языков проектом, по умолчанию установлен английский «en». Tuesday автоматически определяет язык браузера и при наличии подходящей локализации автоматически переключается на нее. В противном случае язык по умолчанию будет включен в английский если он доступен, или на первый язык в этом списке.

Сценарий истории


Структура сценария проекта разделена на несколько элементов. Основным элементом является блок истории, который является частью повествования или сюжетной линии, их последовательность будет определяться выбором игрока (выбор идти влево или вправо может означать, что игрок продолжит текущий блок сюжета или перейдет к другому. блок с альтернативным сюжетом развития). Background image


Аналог css параметра background_image. Предназначен для отображения заднего фона в сцене.

	"background_image": "путь и название файла"

По умолчанию выравнивается по центру стены и пропорционально заполняет весь фон, независимо от размера сцены или самого файла.


Поддерживает локализацию если есть фон для конкретного перевода.

	"background_image": {
			"en":"путь и название файла для английского языка",
			"ja":"путь и название файла для японского языка",
			"ru":"путь и название файла для русского языка"
		}



Dialogs


Массив последовательности диалогов в сцене и всех ее атрибутов, таких как спрайты в сцене art и кнопки выбора Choice.

	"dialogs":[
			{Параметры и элементы реплики 1й диалога},
			{Параметры и элементы реплики 2й диалога},
			{Параметры и элементы реплики 3й диалога}
	]





Элементы диалога


Text


Элемент, содержащий диалоги персонажей и другие тексты. А также элементы для изменения текста text_panel на своей реплике.

	"text": {
		"art":"путь на фоновую картинку кнопки",
		"art_size": ["ширина", "высота"] изображения на кнопке,
		"className": "набор классов стилей css",
		"color_text": "Цвет текста",
		"color_panel": "цвет в форматах",
		"indent_bottom": "отступ текстовой панели от нижнего края",
		"position": [лево, право, верх, низ] позиция отнасительна края сцены, 
		"size_panel": ["ширина", "высота"],
		"size_text": "Размер текста",
		"style": "параметры css и их значения"
		"en":"Boss"
	}

Можно просто указать реплику, в этом случае все недостающие параметры будут взяты из раздела "parameters" / "text_panel"

	"text": "Hello how are you"

Возможно добавить локализованный вариант диалога для указанных языков в разделе "parameters" / "languares"

	"text": {
		"en":"Hello how are you",
		"ja":"こんにちは、元気ですか",
		"ru":"Привет как дела"
		}

В диалоговом окне могут отображаться значения переменной из раздела "parameters" / "variables". Если вы укажете имя переменной в угловых скобках , то вместо этого в тексте появится значение из одноименной переменной. 

{
	"parameters": {
		другие значения
		"variables": [
			"health_points": 100
		],
	}
	"example_block_1": [{
		"dialogs": [{
			"text": "You have <health_points> health points left"
		}]
	}]
}

В этом случае появится фраза "You have 100 health points left"


Иногда бывает, что в одной локации может быть больше или меньше реплик, чем в другой. В этом случае вы можете указать значение 'skip', тогда эта реплика будет автоматически пропущена.

{
	"text": {
		"en":"Hello!",
		"ja":"こんにちは、元気ですか",
		"ru":"Привет!"
	}
},{
	"text": {
		"en":"How are you",
		"ja":"skip",
		"ru":"Kак дела"
	}
}

В этом случае в японской локализации будет пропущена 2-я реплика




Name


Параметр для отображения имени и аватара персонажа в текстовой панели. Эти данные можно указать несколькими способами.


Самый простой вариант просто указать его имя. Если это имя не совпадает с id идентификатором имен в разделе "parameters" / "characters", тогда имя будет отображаться на панели в том виде, в котором вы указали его в этом месте. 

	"name": "Boss"

Возможно задать локализованный вариант имени для указанных языков в разделе "parameters" / "languares"

	"name": {
		"en":"Boss",
		"ja":"ボス",
		"ru":"Босс"
		}

Если вы часто используете одно и то же имя, вы можете указать его в разделе "parameters" / "characters". Это сильно упрощает работу с именами.

{
	"parameters": {
		другие значения
		"characters": {
			"boss": {
				"en":"Boss",
				"ja":"ボス",
				"ru":"Босс"
			}
		}
	}
	"example_block_1": [{
		"dialogs": [{
			"name": "boss"
		}]
	}]
}

Также элементы может изменения name_panel на реплике или имени. Параметры для изменения также можно указать как в "dialogs", так и в массиве "characters".

	"text": {
		"art":"путь на фоновую картинку кнопки",
		"art_size": ["ширина", "высота"] изображения на кнопке,
		"className": "набор классов стилей css",
		"color_text": "Цвет текста",
		"color_panel": "цвет в форматах",
		"indent_bottom": "отступ текстовой панели от нижнего края",
		"position": [лево, право, верх, низ] позиция отнасительна края сцены, 
		"size_panel": ["ширина", "высота"],
		"size_text": "Размер текста",
		"style": "параметры css и их значения"
		"en":"Boss"
	}

Если какой-то элемент будет не указан, то будет взято значение по умолчанию из parameters" / "name_panel". Если имени или его id идентификатора будут отсутствовать, то эта панель отображаться не будет.





Art


Аналог html тега <img>. Предназначен для отображения графических элементов в сцене.

"art": [{		
		"url":"путь и название файла",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"],
		"style": "параметры css и их значения",
		"className": "набор классов стилей css"
	}]

Поддерживает локализацию если есть изображения созданные для конкретного перевода указанного в разделе "parameters" / "languares".

		"url": {
				"en":"путь и название файла для английского языка",
				"ja":"путь и название файла для японского языка",
				"ru":"путь и название файла для русского языка"
			}

Можно добавить в сцену несколько изображений, создав в массиве дополнительные элементы в фигурных скобках со всеми необходимыми параметрами.

"art": [{		
		"url":"Изображени 1",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"]
	},{		
		"url":"Изображени 2",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"]
	}]

Working example "art"

"art": [{		
		"position": ["75%", "50%"],
		"size": ["auto", "90%"],
		"style": "transform: translateX(-50%) translateY(-50%);",
		"className": "example",
		"url": {
				"en":"img/example_en.png",
				"ja":"img/example_ja.png",
				"ru":"img/example_ru.png"
		}
	},{		
		"position": ["25%", "50%"],
		"size": ["auto", "90%"],
		"style": "transform: translateX(-50%) translateY(-50%);",
		"url":"img/character.png"
	}]



Choice


Элемент с кнопками для выбора дальнейшего сюжетного блока и изменения значений в переменных. Аналогичен параметру "buttons" за исключением того, что кнопки выбора появляются в определенном диалоговом окне. 

	"choice": [{
		"go_to": "id блока для перехода",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"],
		"className": "набор классов стилей css",
		"style": "параметры css и их значения",
		"color": "цвет фона",
		"color_text": "Цвет текста",
		"variables": [
			["название переменной", "операция с переменной", значение]
		],
		"text": {
			"en":"Перевод текста на англиский",
			"ja":"Перевод текста на японский",
			"ru":"Перевод текста на русский"
			}
		}
	]

Парметр "go_to" переносит дальнейшее повествование на указанный блок. Если он не указан, то будет продолжен текущий блок.


Параметры "variables" вносят изменения в переменные в момент нажатия на кнопку выбора.


Working example "choice"

{ "name": "inga",
	"text": "which road to take?"
	"choice": [{
		"go_to": "block_left",
		"position": ["25%", "0", "0", "50%"],
		"size": ["256px", "0"],
		"className": "buttons hot_center",
		"text": "<== to the left"
	},{
		"go_to": "block_right",
		"position": ["75%", "0", "0", "50%"],
		"size": ["256px", "0"],
		"className": "buttons hot_center",
		"text": "on the right ==>"
	}]
}



Variables


Параметр для изменения значений из раздела "parameters" / "variables" в конкретном "dialogs" или при выборе "choice"

	"variables": [
		["название переменной", "операция с переменной", значение]
	],

Операция с переменной

"set" - установить значение в переменной ( аналог оператора "=" в JavaScript)

"add" - прибавить значение к уже заданому значению в переменной( аналог оператора "+=" в JavaScript)


Working example "variables"

{
    "parameters": {
		другие значения
		"variables": [
			"experience_points": 80,
			"magic_points": 100,
			"health_points": 100
		],
	},
	"example_block_1": [{
		"dialogs": [{
			"variables": [
				["experience_points", "add", 10],
				["magic_points", "add", -15],
				["health_points", "set", 200]
			]
		}]
	}]
}

При запуске блока "example_block_1"
10 очков опыта будут добавлены к значению "Experience_points",
15 очков будет вычтено из "magic_points",
а "health_points" будет установлено на 200 очков.





Legacy choice


Автоматический переход к другому блоку истории при соблюдении указанных условий в переменных значениях ( parameters / variables )

{
	"legacy_choice": [
		["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"]
	]
}

Важный! Этот элемент не является частью массива "dialogs". И запускать как часть блока рядом с элементами "dialogs". 

"example_block_1": [{
		"dialogs": [{"text": "hello"}]
	},{
		"legacy_choice": [
			["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"]
		]
	}
]

Если условия не совпадают со значениями переменных, то по умолчанию блок продолжится дальше. Но если указать значение "go_to", то в этом случае история перейдет в указанный блок.

{
	"legacy_choice": [
		["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"],
		{"go_to": "example_block_4"}
	]
}

Working example "legacy_choice"

{
    "parameters": {
		другие значения
		"variables": [
			"experience_points": 80,
			"magic_points": 100,
			"health_points": 100
		],
	},
	"example_block_1": [{
		"legacy_choice": [
			["experience_points", "=", 80, "example_block_2"],
			["experience_points", "<", 80, "example_block_3"],
			{"go_to": "example_block_4"}
		]
	}],
	"example_block_2": [{
		"dialogs": [{
			"text": "Awesome!!!"
		}]
	}],
	"example_block_3": [{
		"dialogs": [{
			"text": "you cool"
		}]
	}],
	"example_block_4": [{
		"dialogs": [{
			"text": "hello"
		}]
	}]
}





Параметры элементов



"art" - Аналог css стиля background_image. указывает фоновфй рисунок для кнопки.

		"art":"путь на фоновую картинку кнопки",

"art_size" - Устанавливает размер изображения внутри кнопки, иначе оно отображаться в исходном размере, Задав размер высоты и ширены 100%, размер изображения будет полностью соответствовать размеру кнопки (параметр size). заменив одно из значений на «авто», тогда размер изображения будет пропорционален размеру указанной высоты или ширины кнопки 

		"art_size": ["ширина", "высота"] изображения на кнопке,

"className" - необязательный параметр (аналог атрибута class в html), указывает имя класса стиля из css. Можно задать несколько классов через пробел.

		"className": "параметры css и их значения "

"style" - необязательный параметр (аналог атрибута style в html) персональный css стиль для этого изображения

		"style": "параметры и их значения css"

"position" - Расположение кнопки относительно краев сцены, Например значения [0, 10px, 10px, 0] означают, что кнопка появится в правом верхнем углу с отступом в 10 пикселей.

		"position": [лево, право, верх, низ] позиция отнасительна края сцены,

"url" - Указывает путь к файлу или иному рисурсу (аналог атрибута src в html). 

		"url": "путь и название файла"

"size" - Размер кнопки. Возможно использовать пропорциональные значения относительно размера сцены в процентах. 

		"size": ["ширина", "высота"],

"color" - Цвет фона кнопки в шестнадцатеричном формате hex (#fff #ffffff) или в форматах rgb (255,255,255) и rgba (255,255,255,1). По умолчанию кнопка прозрачная

		"color": "цвет фона",

"color_text" - Цвет текста, необязательный параметр, если не указан то цвет будет по умолчанию, цвет можно установлен в шестнадцатеричном формате hex (#fff #ffffff) или в форматах rgb (255,255,255) и rgba (255,255,255,1).

		"color_text": "Цвет текста",

"indent_text" - Аналог css параметра padding Определяет величину отступа от краев элемента, Можно использовать несколько значений в соответствии с правилами заполнения css. Для разных отступов по вертикали и горизонтали "10px, 5px". Если нужны отступы для каждой стороны или только одной "16px, 5px, 5px, 5px"

		"indent_text": "отступы для текста",

"size_text" - Размер текста внутри элемента

		"size_text": "Размер текста"

"go_to" - Переход к указанному id имени блоку сюжета

		"go_to": "id блока для перехода"






Системные кнопки



"tue_load" - Загрузка стадии истории


"tue_save" - Сохранение стадии истории


"tue_home" - Переход на главный экран указанный в parameters launch_story


"tue_next" - Следующий диалог истории


"tue_back" - Предыдущий диалог истории


"tue_load_autosave" - Загрузка стадии из авто сохранения






Функции tuesday.js



load_story(tip, url) - запуск сценария истории, tip отвечает за тип сценария значение

"file" если сценария в json файл и в значение "url" нужно указать путь к файлу

"data" если сценария в let массиве и в значение "url" нужно указать название массива


go_story() - двигает историю вперед по сюжету. Автоматически переходит к следующему блоку, сцене и диалогу строго по их последовательности.


back_story() - Откатывает историю назад. Автоматически переходит к предыдущему блоку, сцене и диалогу строго по их последовательности.


save_stag(tip) - Сохранение массива текущего состояния истории, такие занижения как текущая сцена диалог блок и все значения из раздела variables в localStorage параметр "tip" название сохранения. Если указать "slot" то в localStorage ото отобразится как tue_slot_data. Занятые названия bookmark для ручного сохранения и auto для автоматического сохранения.


load_stag(tip) - Загрузка массива стадии истории. Параметр "tip" название сохранения, bookmark для ручного сохранения и auto для автоматического сохранения. Или возможно использовать любое другое название которое применялось при использовании функции save_stag(tip)


go_to(go) - Переход в определенный блок истории указав в параметре go его название






События


Ручного сохранения (не работает на autosave)

tuesday.addEventListener('save', function(event) { alert("save") } );

Загрузка ранее сохраненной стадии прогресса

tuesday.addEventListener('load', function(event) { alert("load") } );

Сценарий итосрии загружен успешно

tuesday.addEventListener('script_loaded ', function(event) { alert("script_loaded ") } );

Добавление или вычитаение из определенной переменной, имя переменной с добавленным "_add"

tuesday.addEventListener('variabl_add ', function(event) { alert("change") } );

Изменение определенной переменной, имя переменной с добавленным "_set"

tuesday.addEventListener('variabl_set ', function(event) { alert("set") } );
