TuesDay

visual novel engine

Tuesday бесплатный движок с открытым исходным кодом для создания визуальных новелл, интерактивных историй и текстовых приключенческих игр. Написан на языке JavaScript без использования сторонних библиотек. присутствует поддержка инфраструктуры Apache Cordova для создания нативных приложений для мобильных устройств и настольных компьютеров. Движок использует стандартные элементы Html документа, как div и img, что делает возможным использование почти любых медиа форматов поддерживаемых браузерами, включая векторную графику в формате svg, gif анимации и обширные возможности css стилей.




Структура

Отображение истории на html-странице происходит через элемент div с id tuesday. Параметры отображения можно задать в стиле css или любым другим подходящим способом. Все содержание будет отображаться с учетом параметров этого элемента.


Сценарий истории со всеми данными задается в структуре JSON. Этот независим текстовый формат обмена данными и может быть интерпретирован практически любым другим языком программирования или игровым движком. (это дает возможность переноса истории на другой движок)


Для запуска истории нужно выполнить функцию load_story('тип',файл/массив) с указанием типа данных

data: указание переменной с массивом истории из кода javascript

file: путь к json файлу с историей (может не работать локально, если в браузере включена cross domain защита)




Начало использования

Для работы подойдет любой текстовый редактор или IDE с поддержикой работы с текстовым форматом ASCII

Рекомендуемое программное обеспечения для использования

MS Windows: Notepad++

Mac OS: Xcode

Проблемы запуска и их решения

Если новелла не отобразилось то возможна возникла проблема c ограничением cross domain web security. эту проблему можно решить несколькими способами


1. Решение - Перенести содержимое из json файла в index.html как массив JavaScript

<html>
    <head>
    </head>
    <!--Измениить значениия load_story c ('file','story.json') на ('data',story) -->
    <body onload="load_story('data',story)">
	<div id='tuesday' style='width:100%; height:90vh;'></div>
        <!--Добавит тег script перед tuesday.js --> ;
        <script>
            let story = <!--вставить содержимое из story.json --> ;
        </script>
        <script type="text/javascript" src="tuesday.js"></script>
    </body>
</html>

2. Решение - В некоторых браузерах можно отключить cross domain (не рекомендуется)


FireFox - в адресной строке введите about:config в появившехся настройках найдите пункт security.fileuri.strict_origin_policy и установить ему значение false для отключения или true для включения




Параметры проекта


Базовые параметры и настройки всего проекта задаются в разделе "parameters". От его настроек зависит внешний вид проекта и его некоторый функционал. 

{
    "parameters": {
		"autosave": настройка автосохранения прогресса true / false,
		"buttons": [Кнопки интерфейса и их параметры],
		"characters" : [Глобальные настройки отображения персонажей],
		"languares": [коды языков на которые переведен проект],
		"launch_story": "сцена с которой начинается проект",
		Параметры панели для отображения аватара и имени персонажа
		"name_panel": {
			"size_panel": ["ширина", "высота"],
			"position": [позиция X, позицияY отнасительно text_panel],
			"size_text": "Размер текста",
		},
		Параметры панели для отображения текстов
		"text_panel": {
			"size_panel": ["ширина", "высота"],
			"color_panel": "цвет в форматах hex rgb или rgba",
			"indent_bottom": "отступ текстовой панели от нижнего края",
			"size_text": "Размер текста",
			"dialog_speed": Скорость появления символов диалога
		},
		"title": "Название проекта",
		"key": [Указание кнопок управления для клавиатуры],
		"variables": [набор глобальных переменых использованых в проекте],
}

"autosave" - Параметр отвечает за автоматическое сохранение прогресса истории и значений переменных каждый после запуска диалога. Это позволяет выключить проект и в дальнейшем продолжить его с того же места.

	"autosave": true / false

За продолжения с последнего места отвечает кнопка "load_autosave"




"buttons" - Набор кнопок и их параметры для основного интерфейса как управление диалогами, сохранения и загруска . Они не будут отображаться в сцене указанной в launch_story.

	"buttons": [{
		"name": "Название функцианала или id кнопки",
		"art":"путь на фоновую картинку кнопки",
		"art_size": ["ширина", "высота"] изображения на кнопке,
		"position": [лево, право, верх, низ] позиция отнасительна края сцены, 
		"size": ["ширина", "высота"],
		"color": "цвет в форматах hex rgb или rgba",
		"className": "набор классов стилей css",
		"style": "параметры css и их значения",
		"indent_text": "отступы для текста",
		"size_text": "Размер текста"
	}]

"name" - идентификатор кнопки (аналог id в html), он будет определять функциональность этой кнопки. Для интерфейса понадобятся следующий набор кнопок:

"tue_load" - Загрузка стадии истории

"tue_home" - Переход на главный экран указанный в parameters launch_story

"tue_next" - Следующий диалог истории

Больше функцианала в разделе System Buttons

		"art":"путь на фоновую картинку кнопки",

"art_size" - Устанавливает размер изображения внутри кнопки, иначе оно отображаться в исходном размере, обязательно должны быть указаны единицы измерения принятые в CSS. Задав размер высоты и ширены 100%, размер изображения будет полностью соответствовать размеру кнопки (параметр size). заменив одно из значений на «авто», тогда размер изображения будет пропорционален размеру указанной высоты или ширины кнопки

Графические элементы



Element "background_image"


Аналог css параметра background_image. Предназначен для отображения заднего фона в сцене.

	"background_image": "путь и название файла"

По умолчанию выравнивается по центру стены и пропорционально заполняет весь фон, независимо от размера сцены или самого файла.


Поддерживает локализацию если есть фон для конкретного перевода.

	"background_image": {
			"en":"путь и название файла для английского языка",
			"ja":"путь и название файла для японского языка",
			"ru":"путь и название файла для русского языка"
		}



Element "art"


Аналог html тега <img>. Предназначен для отображения графических элементов в сцене.

"art": [{		
		"url":"путь и название файла",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"],
		"style": "параметры css и их значения",
		"className": "набор классов стилей css"
	}]

"url" - Указывает путь к файлу (аналог src в html). 

		"url": "путь и название файла"

Поддерживает локализацию если есть изображения созданные для конкретного перевода.

		"url": {
				"en":"путь и название файла для английского языка",
				"ja":"путь и название файла для японского языка",
				"ru":"путь и название файла для русского языка"
			}

"size" - Устанавливает размер изображения в сцене, иначе оно отображаться в исходном размере, обязательно должны быть указаны единицы измерения принятые в CSS 

		"size": ["ширина", "высота"]

"position" - Устанавливает позицию изображения в сцене, обязательно должны быть указаны единицы измерения принятые в CSS

		"position": [позиция X, позицияY]

"style" - необязательный параметр (аналог style в html) персональный css стиль для этого изображения

		"style": "параметры и их значения css"

"className" - необязательный параметр (аналог class в html), указывает имя класса стиля из css. Можно задать несколько классов через пробел.

		"className": "параметры css и их значения "

Можно добавить в сцену несколько изображений, создав в массиве дополнительные элементы в фигурных скобках со всеми необходимыми параметрами.

"art": [{		
		"url":"Изображени 1",

		"size": ["ширина", "высота"]
	},{		
		"url":"Изображени 2",
		"position": [позиция X, позицияY],
		"size": ["ширина", "высота"]
	}]

Функции tuesday.js


load_story(tip, url) - запуск сценария истории, tip отвечает за тип сценария значение

"file" если сценария в json файл и в значение "url" нужно указать путь к файлу

"data" если сценария в let массиве и в значение "url" нужно указать название массива


go_story() - двигает историю вперед по сюжету. Автоматически переходит к следующему блоку, сцене и диалогу строго по их последовательности.


back_story() - Откатывает историю назад. Автоматически переходит к предыдущему блоку, сцене и диалогу строго по их последовательности.


save_stag(tip) - Сохранение массива текущего состояния истории, такие занижения как текущая сцена диалог блок и все значения из раздела variables в localStorage параметр "tip" название сохранения. Если указать "slot" то в localStorage ото отобразится как tue_slot_data. Занятые названия bookmark для ручного сохранения и auto для автоматического сохранения.


load_stag(tip) - Загрузка массива стадии истории. Параметр "tip" название сохранения, bookmark для ручного сохранения и auto для автоматического сохранения. Или возможно использовать любое другое название которое применялось при использовании функции save_stag(tip)


go_to(go) - Переход в определенный блок истории указав в параметре go его название
